# 数据源配置指南

## 配置方式

数据源配置存储在 PostgreSQL 数据库中，可通过以下方式操作：

### 1. 使用 CLI 管理脚本

```bash
# 进入容器
docker exec -it clue-collector bash

# 列出分组
python scripts/manage_sources.py list-groups

# 添加分组
python scripts/manage_sources.py add-group --name "热榜数据源" --interval 5 --active

# 列出数据源
python scripts/manage_sources.py list-sources

# 添加数据源（配置为JSON字符串）
python scripts/manage_sources.py add-source \
  --group-id "00000000-0000-0000-0000-000000000001" \
  --name "微博热搜" \
  --type hotlist \
  --config '{"url":"https://weibo.com/ajax/side/hotSearch","method":"GET","parse_type":"json","parse_rules":{"container":"data.realtime","title":"note","rank":"rank","heat":"num"}}' \
  --priority 9

# 添加数据源（配置为JSON文件）
python scripts/manage_sources.py add-source \
  --group-id "xxx" \
  --name "知乎热榜" \
  --type hotlist \
  --config ./configs/zhihu.json

# 查看示例配置
python scripts/manage_sources.py example-configs
```

### 2. 运行示例配置脚本

```bash
# 添加预置的热榜数据源（微博、知乎、百度、抖音、B站）
docker exec -it clue-collector python scripts/add_sources.py
```

### 3. 直接操作数据库

```bash
# 连接数据库
docker exec -it clue-collector-postgres psql -U postgres -d clue_collector

# 查看分组
SELECT id, name, collect_interval, is_active FROM source_groups;

# 查看数据源
SELECT id, name, type, config, priority, status FROM data_sources;
```

## 配置结构

### 分组配置 (source_groups)

| 字段 | 说明 |
|------|------|
| name | 分组名称 |
| collect_interval | 采集频率（分钟） |
| is_active | 是否启用 |

### 数据源配置 (data_sources)

| 字段 | 说明 | 可选值 |
|------|------|--------|
| group_id | 所属分组ID | UUID |
| name | 数据源名称 | 字符串 |
| type | 数据源类型 | hotlist/account/video/custom |
| collector_type | 采集器类型 | configurable/plugin |
| config | 采集配置 | JSON对象 |
| priority | 优先级 | 1-10（越高越优先） |
| is_active | 是否启用 | true/false |

## 采集配置格式 (config)

### 配置化采集器

适用于大多数热榜、API 类数据源：

```json
{
  "url": "目标URL",
  "method": "GET/POST",
  "parse_type": "json/css/xpath",
  "parse_rules": {
    "container": "数据容器路径",
    "title": "标题字段",
    "rank": "排名字段（可选）",
    "heat": "热度字段（可选）",
    "link": "链接字段",
    "author": "作者字段（可选）",
    ...
  },
  "headers": {
    "User-Agent": "...",
    "Referer": "..."
  },
  "use_playwright": false  // 是否需要动态渲染
}
```

#### parse_type 类型

| 类型 | 适用场景 | 示例 |
|------|----------|------|
| json | JSON API | 知乎、微博API、B站API |
| css | HTML页面 | 百度热搜、大部分网站 |
| xpath | 复杂HTML | 需要精确定位时 |

#### parse_rules 字段语法

**JSON路径**：使用点号分隔，如 `data.realtime`、`target.title`

**CSS选择器**：标准CSS语法，如 `.hot-list li`、`#content .item`

**链接提取**：使用 `@href`，如 `a@href`

**模板链接**：需要拼接时使用 `{"template": "https://xxx?q={word}"}`

### 需要动态渲染的页面

部分网站需要 JavaScript 渲染，设置 `use_playwright: true`：

```json
{
  "url": "https://www.douyin.com/hot",
  "parse_type": "css",
  "parse_rules": {...},
  "use_playwright": true
}
```

## 示例配置

### 微博热搜（JSON API）

```json
{
  "url": "https://weibo.com/ajax/side/hotSearch",
  "method": "GET",
  "parse_type": "json",
  "parse_rules": {
    "container": "data.realtime",
    "title": "note",
    "rank": "rank",
    "heat": "num",
    "link": {"template": "https://s.weibo.com/weibo?q={word}"},
    "word": "word"
  },
  "headers": {
    "User-Agent": "Mozilla/5.0 Chrome/120.0.0.0",
    "Referer": "https://weibo.com"
  }
}
```

### 知乎热榜（JSON API）

```json
{
  "url": "https://www.zhihu.com/api/v3/feed/topstory/hot-lists/total",
  "method": "GET",
  "parse_type": "json",
  "parse_rules": {
    "container": "data",
    "title": "target.title",
    "heat": "detail_text",
    "link": "target.url",
    "excerpt": "target.excerpt"
  }
}
```

### B站热门（JSON API）

```json
{
  "url": "https://api.bilibili.com/x/web-interface/popular",
  "method": "GET",
  "parse_type": "json",
  "parse_rules": {
    "container": "data.list",
    "title": "title",
    "author": "owner.name",
    "likes": "stat.like",
    "link": {"template": "https://www.bilibili.com/video/{bvid}"},
    "bvid": "bvid",
    "cover": "pic"
  }
}
```

### CSS选择器示例

```json
{
  "url": "https://example.com/hot",
  "method": "GET",
  "parse_type": "css",
  "parse_rules": {
    "container": ".hot-list li",
    "title": ".title",
    "rank": ".rank",
    "heat": ".heat-value",
    "link": "a@href"
  }
}
```

## 插件化采集器

对于需要登录、Cookie管理、复杂反爬的数据源，使用 `collector_type: plugin`：

```json
{
  "collector_type": "plugin",
  "config": {
    "plugin_name": "weibo_account",
    "account_id": "xxx",
    "cookies_required": true
  }
}
```

插件需要单独开发，继承 `BaseCollector` 接口。

## 采集流程

1. 调度器每分钟扫描分组
2. 根据分组频率判断是否需要采集
3. 将活跃数据源加入优先级队列
4. Worker 从队列取出任务执行
5. 采集结果存入 clues 表
6. 日志记录存入 collect_logs 表

## 常见问题

**Q: 如何调试采集配置？**
A: 可以直接访问目标 URL，查看响应格式，根据结构调整 parse_rules。

**Q: 采集失败怎么办？**
A: 查看 collect_logs 表的 error_message 字段，常见原因：
- 配置规则不匹配
- 需要登录/Cookie
- 反爬限制（429/403）
- 网络超时

**Q: 如何处理反爬？**
A: 系统内置反爬模块：
- IP轮换（需要配置外部代理池）
- UA轮换（自动）
- Cookie池（需要预置登录态）
- 频率自适应（检测到限制自动降频）
