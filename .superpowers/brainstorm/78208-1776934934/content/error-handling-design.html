异常处理模块设计

错误分类 + 智能重试 + 降级策略 + 日志告警

异常处理架构
🔍 错误类型分类
错误类型触发条件处理策略
network_timeout请求超时重试3次，每次更换IP
connection_error连接失败重试2次，检查代理可用性
blocked_ip403/429状态码立即切换IP，降频采集
parse_error解析失败记录日志，通知管理员检查规则
cookie_expired登录态失效切换Cookie池中下一个
source_unavailable源站不可访问暂停数据源，等待恢复检测
🔁 智能重试策略 
retry_config:
  max_retries: 3              # 最大重试次数
  retry_delay:                # 重试延迟（秒）
    initial: 5                # 初始延迟
    multiplier: 2             # 延迟倍数
    max: 60                   # 最大延迟

  retry_on_errors:            # 需重试的错误类型
    - network_timeout
    - connection_error
    - blocked_ip
    - cookie_expired

  no_retry_on:                # 不重试的错误类型
    - parse_error             # 规则问题，重试无效
    - source_unavailable      # 源站问题，需人工介入
延迟策略： 5s → 10s → 20s（指数退避）
⏬ 降级策略
1. IP降级
连续3次IP被封 → 标记IP不可用 → 切换新IP → 通知代理池服务
2. 频率降级
检测到429 → 采集频率降低50% → 连续成功5次后逐步恢复
3. 数据源降级
连续5次失败 → 数据源状态改为 error → 暂停采集 → 通知管理员

🔔 告警机制
告警触发条件：
  • 数据源连续5次采集失败
  • 数据源状态变更为 paused
  • 解析规则失效（parse_error 连续3次）
  • 代理池可用IP低于阈值（<10）
  • Cookie池全部失效
告警方式（可配置）：
  • 日志记录（结构化JSON日志）
  • 邮件通知
  • Webhook（对接企业微信/钉钉）
  • Slack消息
🧩 异常处理模块代码结构 
class ErrorHandler:
    """异常处理器"""

    def __init__(self, config: RetryConfig):
        self.config = config
        self.alerter: Alerter
        self.source_status_manager: SourceStatusManager

    async def handle(self, error: Exception, context: CollectContext) -> HandleResult:
        """处理采集异常"""
        error_type = self.classify_error(error)
        source_id = context.source_id

        if error_type in self.config.no_retry_on:
            # 不重试，直接记录并告警
            await self.log_error(source_id, error_type, error)
            await self.alerter.notify(source_id, error_type, error)
            await self.source_status_manager.pause(source_id, error_type)
            return HandleResult(action="abort", success=False)

        # 尝试重试
        for attempt in range(self.config.max_retries):
            delay = self.calculate_delay(attempt)
            await asyncio.sleep(delay)

            # 应用降级措施
            await self.apply_degradation(error_type, context)

            # 重新执行采集
            result = await self.retry_collect(context)
            if result.success:
                return HandleResult(action="retry_success", success=True, attempts=attempt+1)

        # 重试全部失败
        await self.log_error(source_id, error_type, error, retry_count=self.config.max_retries)
        await self.alerter.notify(source_id, error_type, error, retry_count=self.config.max_retries)
        await self.source_status_manager.pause(source_id, error_type)
        return HandleResult(action="abort", success=False)

    def classify_error(self, error: Exception) -> str:
        """错误分类"""
        if isinstance(error, asyncio.TimeoutError):
            return "network_timeout"
        elif isinstance(error, ConnectionError):
            return "connection_error"
        elif isinstance(error, BlockedError):  # 403/429
            return "blocked_ip"
        elif isinstance(error, ParseError):
            return "parse_error"
        elif isinstance(error, CookieExpiredError):
            return "cookie_expired"
        else:
            return "source_unavailable"

    def calculate_delay(self, attempt: int) -> int:
        """计算重试延迟（指数退避）"""
        delay = self.config.retry_delay.initial * (self.config.retry_delay.multiplier ** attempt)
        return min(delay, self.config.retry_delay.max)

设计要点： 错误分类决定处理策略，可恢复错误智能重试，不可恢复错误立即告警。降级措施逐步生效，避免雪崩效应。
