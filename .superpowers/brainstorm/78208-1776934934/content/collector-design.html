采集器设计

配置化采集器 + 插件化采集器 混合架构

采集引擎模块结构
🔧 配置化采集器
适用于：热榜类数据源（规则标准化）
配置示例：
{
  "url": "https://weibo.com/hot",
  "method": "GET",
  "parse_type": "css",  // css/json/xpath
  "parse_rules": {
    "container": ".hot-list li",
    "title": ".title",
    "rank": ".rank",
    "heat": ".heat-value",
    "link": "a[href]"
  },
  "pagination": {
    "enabled": false
  }
}
支持的解析类型：
CSS选择器 JSON API XPath
适用数据源：
微博热搜、知乎热榜、抖音热点、今日头条、百度热搜、B站热门等
🧩 插件化采集器
适用于：复杂数据源（需要特殊处理）
插件接口定义：
class BaseCollector(ABC):
    name: str           # 插件名称
    version: str        # 版本号
    supported_types: [] # 支持的数据类型

    async def collect(
        self,
        source: DataSource,
        anti_crawl: AntiCrawlModule
    ) -> List[ClueData]:
        """执行采集，返回线索数据"""

    async def validate(
        self,
        config: dict
    ) -> bool:
        """验证配置是否有效"""
插件需要处理：
登录态保持 Cookie管理 动态渲染 反爬对抗
适用数据源：
微信公众号、微博大V、抖音账号、小红书博主、自定义API源
🕷️ Scrapling 框架集成
配置化采集器使用：
from scrapling import FetchAdaptor

# 自动适应页面变化
page = FetchAdaptor.auto_match(
    url=config.url,
    auto_match=True  # 自动匹配元素
)
items = page.css(config.parse_rules['container'])
插件化采集器使用：
from scrapling import PlaywrightAdaptor

# 处理动态渲染页面
page = PlaywrightAdaptor.auto_match(
    url=url,
    stealth=True,  # 隐身模式
    wait_for='selector'
)

采集器选择规则： 新增数据源时，若解析规则可标准化描述 → 配置化采集器；若需要登录/动态渲染/特殊反爬 → 插件化采集器。
